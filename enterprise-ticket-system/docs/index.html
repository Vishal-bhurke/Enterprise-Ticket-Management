Ticket Lifecycle in 6 Steps

From submission to resolution — automated at every stage.

1

User submits a ticket

The end user fills in the title, type, priority, category, and description. The system assigns a unique TICK-##### number instantly via database trigger.

2

SLA policy is applied automatically

The system matches the ticket's priority to an SLA policy. Response and resolution deadlines are calculated — accounting for business hours and the priority's SLA multiplier. The clock starts immediately.

3

Automation rules fire

If a matching automation rule exists (e.g., "on ticket_created where priority = critical"), it fires instantly — assigning the ticket to the right queue, notifying the team lead, and adding relevant tags.

4

Agent works the ticket

The assigned agent picks up the ticket, updates its status to In Progress, adds internal notes for colleagues, and communicates with the requester via comments. The SLA timer shows time remaining in real time.

5

Ticket is resolved — SLA clock stops

Agent sets status to Resolved. The database trigger records resolved_at and marks the SLA resolution event as met. The requester receives an automatic notification.

6

Audit trail is permanent

Every status change, reassignment, and comment is recorded in the immutable audit log — automatically, by database triggers. Admins can view the before/after diff for any change at any time.

Deploy in under 60 minutes

Three steps from zero to production. No Docker. No servers. No DevOps team required.

1

Set up Supabase

Create a project and run the 10 migration files in order

Option A — SQL Editor (paste and run each file)

-- Run these in Supabase SQL Editor, in order: 001_core_tables.sql 002_ticket_tables.sql 003_workflow_tables.sql 004_sla_tables.sql 005_automation_tables.sql 006_notification_tables.sql 007_audit_tables.sql 008_rls_policies.sql 009_functions_triggers.sql 010_seed_data.sql

Option B — Supabase CLI

supabase login supabase link --project-ref YOUR_PROJECT_REF supabase db push
2

Configure environment

Add your Supabase URL and anon key to the environment files

src/environments/environment.prod.ts

export const environment = { production: true, supabase: { url: 'https://YOUR_PROJECT.supabase.co', anonKey: 'YOUR_ANON_KEY_HERE', }, appName: 'Enterprise Ticket System', version: '1.0.0', };
3

Build and deploy to Netlify

One command to build, then drag the output folder to Netlify

# Install dependencies npm install # Build for production npx ng build --configuration=production # Output is in: dist/enterprise-ticket-system/browser/

Deploy to Netlify

  1. Go to app.netlify.com
  2. Drag the dist/enterprise-ticket-system/browser/ folder onto the drop zone
  3. Done — live in seconds ✓
